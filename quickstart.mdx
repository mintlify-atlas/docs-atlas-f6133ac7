This guide will help you get up and running with the Rihsac API. You'll learn how to register a user, authenticate, and complete the onboarding process.

## Prerequisites

Before you begin, make sure you have:

- A Rihsac API base URL (e.g., `https://api.rihsac.com`)
- An HTTP client (cURL, Postman, or code in your preferred language)
- Basic understanding of REST APIs and JWT authentication

<Note>
All API requests should be made to `/api` endpoints (e.g., `https://api.rihsac.com/api/register`)
</Note>

## Step 1: Register a new user

Create a new user account by making a POST request to the `/api/register` endpoint.

<CodeGroup>

```bash cURL
curl -X POST https://api.rihsac.com/api/register \
  -H "Content-Type: application/json" \
  -d '{
    "firstname": "John",
    "lastname": "Doe",
    "email": "john.doe@example.com",
    "password": "secure_password123",
    "state_id": 1,
    "phone_number": "+1234567890"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.rihsac.com/api/register', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    firstname: 'John',
    lastname: 'Doe',
    email: 'john.doe@example.com',
    password: 'secure_password123',
    state_id: 1,
    phone_number: '+1234567890'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.post(
    'https://api.rihsac.com/api/register',
    json={
        'firstname': 'John',
        'lastname': 'Doe',
        'email': 'john.doe@example.com',
        'password': 'secure_password123',
        'state_id': 1,
        'phone_number': '+1234567890'
    }
)

print(response.json())
```

</CodeGroup>

<Warning>
Store the JWT token securely. You'll need it for all subsequent authenticated requests.
</Warning>

## Step 2: Login (if needed)

If you already have an account, you can login to get a JWT token:

```bash
curl -X POST https://api.rihsac.com/api/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john.doe@example.com",
    "password": "secure_password123"
  }'
```

## Step 3: Complete onboarding

After registration, you must complete the onboarding process. The user's `onboarding_stage` field tracks progress.

### 3.1 Verify email address

Check your email for a verification code and submit it:

```bash
curl -X POST https://api.rihsac.com/api/email/verify \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "code": "123456"
  }'
```

### 3.2 Update and verify phone number

If you need to update your phone number:

```bash
curl -X POST https://api.rihsac.com/api/phone/update \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "firstname": "John",
    "lastname": "Doe",
    "state_id": 1,
    "phone_number": "+1234567890"
  }'
```

Then verify the phone number with the SMS code:

```bash
curl -X POST https://api.rihsac.com/api/phone/verify \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "code": "654321"
  }'
```

### 3.3 Set transaction PIN

Create a 4-digit transaction PIN for secure operations:

```bash
curl -X POST https://api.rihsac.com/api/transaction-pin \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "pin": "1234",
    "pin_confirmation": "1234"
  }'
```

### 3.4 Create username

Choose a unique username:

```bash
curl -X POST https://api.rihsac.com/api/username/update \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "username": "johndoe"
  }'
```

## Step 4: Make your first profile request

Once onboarded, you can start using the API. For example, add guarantor information:

```bash
curl -X POST https://api.rihsac.com/api/profile/guarantor \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "firstname": "Jane",
    "lastname": "Smith",
    "phone_number": "+1234567891",
    "email": "jane.smith@example.com",
    "setting_id": 1
  }'
```

## Onboarding stages

The user's `onboarding_stage` progresses through these values:

| Stage | Description |
|-------|-------------|
| `email_verification` | User needs to verify their email |
| `profile_update` | User needs to update profile information |
| `phone_verification` | User needs to verify phone number |
| `username_creation` | User needs to create a username |
| `transaction_pin` | User needs to set a transaction PIN |
| `completed` | Onboarding is complete |

## Common issues

<Accordion title="'Unauthenticated' error">
Make sure you're including the JWT token in the Authorization header:
```
Authorization: Bearer YOUR_JWT_TOKEN
```
</Accordion>

<Accordion title="Validation errors">
Check that all required fields are included and meet the validation requirements (e.g., firstname max 20 chars, password min 6 chars).
</Accordion>

<Accordion title="Email verification code not received">
Use the resend endpoint:
```bash
curl -X POST https://api.rihsac.com/api/email/resend \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```
</Accordion>
