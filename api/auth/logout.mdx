## Endpoint

```
POST /api/logout
```

Logout the currently authenticated user. This endpoint invalidates the JWT token and logs the logout activity.

<Note>
This endpoint requires authentication. Include the JWT token in the Authorization header.
</Note>

## Authentication

**Required:** Yes  
**Middleware:** `jwt.user`

Include the JWT token in the request header:

```
Authorization: Bearer {your_jwt_token}
```

## Request

No request body parameters required.

## Response

<ResponseField name="message" type="string">
  Confirmation message indicating successful logout
</ResponseField>

## Logout Process

<Expandable title="What happens during logout?">

1. **Token Validation**: The JWT token is validated through the `jwt.user` middleware

2. **Session Invalidation**: 
   - The user's authentication session is terminated
   - The JWT token is invalidated and cannot be used for future requests

3. **Activity Logging**: 
   - Logout action is recorded in the activity log
   - Activity type: `LOGOUT`
   - Includes user ID and timestamp

4. **Response**: Confirmation message is returned

</Expandable>

## Security Notes

- After logout, the JWT token becomes invalid and cannot be reused
- All active sessions for the user are terminated
- Client applications should remove stored tokens after receiving a successful logout response
- Logout activity is tracked in the system activity log for security auditing

## Next Steps

After logout, users must:
1. Login again using the `/api/login` endpoint to obtain a new JWT token
2. Client applications should redirect to the login page
3. Clear any cached user data or tokens from local storage
