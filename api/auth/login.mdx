## Endpoint

```
POST /api/login
```

Authenticate a user with email and password credentials. Returns a JWT token for accessing protected endpoints.

## Request Body

<ParamField body="email" type="string" required>
  User's email address
  
  **Validation:** Must be a valid email format
</ParamField>

<ParamField body="password" type="string" required>
  User's password
  
  **Validation:** Minimum 6 characters
</ParamField>

## Response

<ResponseField name="status" type="string">
  Response status - `success` or `failure`
</ResponseField>

<ResponseField name="message" type="string">
  Response message
</ResponseField>

<ResponseField name="data" type="string">
  JWT access token for authenticated requests
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl -X POST https://api.rihsac.com/api/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "password123"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.rihsac.com/api/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'password123'
  })
});

const data = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.rihsac.com/api/login',
    json={
        'email': 'user@example.com',
        'password': 'password123'
    }
)

data = response.json()
```

```php PHP
$curl = curl_init();

curl_setopt_array($curl, [
    CURLOPT_URL => 'https://api.rihsac.com/api/login',
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_POST => true,
    CURLOPT_HTTPHEADER => ['Content-Type: application/json'],
    CURLOPT_POSTFIELDS => json_encode([
        'email' => 'user@example.com',
        'password' => 'password123'
    ])
]);

$response = curl_exec($curl);
curl_close($curl);

$data = json_decode($response);
```

</CodeGroup>

## Status Codes

<ResponseField name="200" type="Success">
  User authenticated successfully. JWT token returned in the response data.
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Authentication failed. Invalid credentials or unverified email account.
</ResponseField>

<ResponseField name="422" type="Validation Error">
  Request validation failed. Check the error field for specific validation messages.
</ResponseField>

## Notes

- The returned JWT token should be included in subsequent requests using the `Authorization: Bearer {token}` header
- Login activity is automatically logged in the system activity log
- The token expiration time is configured in the JWT settings
