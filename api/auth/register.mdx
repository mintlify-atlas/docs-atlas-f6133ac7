## Endpoint

```
POST /api/register
```

Register a new user account. Upon successful registration, the user is automatically logged in and receives a JWT token. A verification email is sent to the provided email address.

## Request Body

<ParamField body="firstname" type="string" required>
  User's first name
  
  **Validation:** Required, maximum 20 characters
</ParamField>

<ParamField body="lastname" type="string" required>
  User's last name
  
  **Validation:** Required, maximum 20 characters
</ParamField>

<ParamField body="email" type="string" required>
  User's email address
  
  **Validation:** Must be a valid email format and unique in the system
</ParamField>

<ParamField body="password" type="string" required>
  User's password
  
  **Validation:** Minimum 6 characters
</ParamField>

<ParamField body="state_id" type="integer" required>
  ID of the user's state/region
  
  **Validation:** Required, must be a valid integer
</ParamField>

<ParamField body="phone_number" type="string" required>
  User's phone number
  
  **Validation:** Required
</ParamField>

## Response

<ResponseField name="status" type="string">
  Response status - `success` or `failure`
</ResponseField>

<ResponseField name="message" type="string">
  Response message describing the result
</ResponseField>

<ResponseField name="data" type="string">
  JWT access token for the newly created and authenticated user
</ResponseField>

## Post-Registration Flow

<Expandable title="What happens after registration?">

1. **Account Creation**: User account is created with the following defaults:
   - `uuid`: Automatically generated UUID v4
   - `cashir_id`: Random 10-digit identifier
   - `onboarding_stage`: Set to `EMAIL_VERIFY`
   - `role`: Set to `USER`
   - `status`: Set to `enabled`

2. **Automatic Login**: User is automatically authenticated and receives a JWT token

3. **Email Verification**: 
   - A verification code is generated
   - An email is sent to the provided email address
   - User must verify email before full account access

4. **Activity Logging**: Registration is logged in the system activity log

5. **Next Steps**: User should:
   - Verify email address using `/email/verify` endpoint
   - Complete profile information
   - Set up additional security features

</Expandable>

## Notes

- The returned JWT token can be used immediately for authenticated requests
- Email verification is required to complete the onboarding process
- The onboarding stage tracks user progress through account setup
- Password is automatically hashed before storage
- Activity is logged with action type `REGISTER`
